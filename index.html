Git e GitHub - Conceitos e Práticas

O que é Git?

Git é um sistema de controle de versão distribuído, criado por Linus Torvalds em 2005. Ele permite que desenvolvedores acompanhem mudanças em arquivos e coordenem o trabalho em projetos entre múltiplas pessoas.

O que é GitHub?

GitHub é uma plataforma de hospedagem de código-fonte que utiliza o Git para controle de versão. Oferece recursos adicionais como interface web, colaboração, gerenciamento de projetos e integração contínua.

Principais Conceitos:

  • Repositório: Um diretório que contém todos os arquivos do projeto e o histórico completo de mudanças.
  • Commit: Um snapshot do projeto em um momento específico, com uma mensagem descritiva das mudanças.
  • Branch: Uma linha de desenvolvimento independente que permite trabalhar em features sem afetar o código principal.
  • Merge: Processo de combinar mudanças de diferentes branches em uma única linha de desenvolvimento.
  • Pull Request: Solicitação para incorporar mudanças de um branch para outro, permitindo revisão de código.
  • Clone: Cópia completa de um repositório remoto para o ambiente local de desenvolvimento.
  • Fork: Cópia de um repositório para sua conta, permitindo contribuições independentes.
  • Remote: Referência a um repositório hospedado em outro local (como GitHub).

Comandos Git Essenciais:

Configuração inicial:

git config --global user.name "Seu Nome"
git config --global user.email "seu@email.com"

Comandos básicos:

git init

Inicializa um novo repositório Git

git clone <url>

Clona um repositório remoto

git status

Mostra o status dos arquivos

git add <arquivo>

Adiciona arquivo para o staging area

git add .

Adiciona todos os arquivos modificados

git commit -m "mensagem"

Cria um commit com mensagem

git push origin <branch>

Envia commits para o repositório remoto

git pull

Baixa e integra mudanças do repositório remoto

git log

Mostra o histórico de commits
