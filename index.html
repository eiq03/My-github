Git e GitHub - Conceitos e Práticas

O que é Git?

Git é um sistema de controle de versão distribuído, criado por Linus Torvalds em 2005. Ele permite que desenvolvedores acompanhem mudanças em arquivos e coordenem o trabalho em projetos entre múltiplas pessoas.

O que é GitHub?

GitHub é uma plataforma de hospedagem de código-fonte que utiliza o Git para controle de versão. Oferece recursos adicionais como interface web, colaboração, gerenciamento de projetos e integração contínua.

Principais Conceitos:

  • Repositório: Um diretório que contém todos os arquivos do projeto e o histórico completo de mudanças.
  • Commit: Um snapshot do projeto em um momento específico, com uma mensagem descritiva das mudanças.
  • Branch: Uma linha de desenvolvimento independente que permite trabalhar em features sem afetar o código principal.
  • Merge: Processo de combinar mudanças de diferentes branches em uma única linha de desenvolvimento.
  • Pull Request: Solicitação para incorporar mudanças de um branch para outro, permitindo revisão de código.
  • Clone: Cópia completa de um repositório remoto para o ambiente local de desenvolvimento.
  • Fork: Cópia de um repositório para sua conta, permitindo contribuições independentes.
  • Remote: Referência a um repositório hospedado em outro local (como GitHub).

Comandos Git Essenciais:

Configuração inicial:

git config --global user.name "Seu Nome"
git config --global user.email "seu@email.com"

Comandos básicos:

git init

Inicializa um novo repositório Git

git clone <url>

Clona um repositório remoto

git status

Mostra o status dos arquivos

git add <arquivo>

Adiciona arquivo para o staging area

git add .

Adiciona todos os arquivos modificados

git commit -m "mensagem"

Cria um commit com mensagem

git push origin <branch>

Envia commits para o repositório remoto

git pull

Baixa e integra mudanças do repositório remoto

git log

Mostra o histórico de commits

2. Fluxo de Trabalho Git/GitHub

Fluxo Básico de Desenvolvimento

Passo 1: Clone do Repositório

Primeiro passo é clonar o repositório para seu ambiente local:

git clone https://github.com/usuario/My_github.git
cd My_github

Passo 2: Criação de Feature Branch

Crie uma nova branch para desenvolver a funcionalidade:

git checkout -b feat/conceitos-git-github

Passo 3: Desenvolvimento

Faça as modificações necessárias nos arquivos do projeto. Crie ou edite arquivos HTML, CSS, JavaScript, etc.

Passo 4: Verificação das Mudanças

git status
git diff

Passo 5: Staging e Commit

Adicione as mudanças e crie um commit:

git add .
git commit -m "feat: adiciona página de conceitos Git/GitHub"

Passo 6: Push para o Repositório Remoto

git push origin feat/conceitos-git-github

Passo 7: Criação de Pull Request

No GitHub, crie um Pull Request para integrar sua branch à branch principal (main/master)

Passo 8: Review e Merge

Após aprovação, faça o merge do Pull Request

Passo 9: Limpeza Local

git checkout main
git pull origin main
git branch -d feat/conceitos-git-github

Fluxo para Múltiplas Features

Para cada nova feature (conceitos, fluxo, conflitos), repita o processo:

  1. git checkout main
  2. git pull origin main
  3. git checkout -b feat/nova-feature
  4. Desenvolver a feature
  5. git add . && git commit -m "mensagem"
  6. git push origin feat/nova-feature
  7. Criar Pull Request no GitHub

Boas Práticas

  • Use mensagens de commit descritivas e padronizadas
  • Mantenha commits pequenos e focados
  • Sempre teste antes de fazer push
  • Use branches para cada feature
  • Mantenha a branch main sempre estável
  • Faça pull requests para revisão de código
